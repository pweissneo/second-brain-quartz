# Progressive Summarization vs Zettelkasten

Two powerful note-taking methodologies with different philosophies and use cases.

## Progressive Summarization

See [[Progressive Summarization]] for full details.

**Philosophy:** Distill information through multiple passes (bold → bullets → own words → links)
**Best for:** Books, articles, videos (consumable content), passive input processing
**Creator:** Tiago Forte (PARA method)

## Zettelkasten

**Philosophy:** Create atomic, interconnected notes

**Process:**
1. Extract one idea per note
2. Write in your own words
3. Link to related notes
4. Let emergence happen

**Best for:**
- Research and writing
- Active knowledge creation
- Building a lattice of ideas

**Creator:** Niklas Luhmann

## Comparison

| Aspect | Progressive Summarization | Zettelkasten |
|--------|---------------------------|---------------|
| Unit | Summary | Atomic note |
| Focus | Compression | Connection |
| Input | Consumable content | Any idea |
| Output | Quick recall | New insights |
| Time | Fast (15 min/chapter) | Slow (1 note/idea) |

## When to Use Each

**Use Progressive Summarization when:**
- Processing books or articles
- Need quick reference summaries
- Building a reference library

**Use Zettelkasten when:**
- Doing research or writing
- Wanting to generate new ideas
- Building a knowledge network

## The Best of Both Worlds

Use **Progressive Summarization** for initial capture, then convert summaries to **atomic Zettelkasten notes** for linking.

1. Highlight → Bold → Bullet (Progressive Summarization)
2. Extract atomic idea → Write in own words (Zettelkasten)
3. Link to related notes (both)
