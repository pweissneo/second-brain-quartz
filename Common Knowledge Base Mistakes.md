# Common Knowledge Base Mistakes

Errors that derail your knowledge management system.

## Starting Too Complex

### Mistake
Setting up elaborate templates, folders, tagging systems before writing any notes.

### Solution
Start simple. Just write notes and link them.

---

## Obsessing Over Tools

### Mistake
Switching apps, trying plugins, customizing themes instead of capturing knowledge.

### Solution
Tools serve knowledge, not the reverse.

---

## Creating Orphan Notes

### Mistake
Writing notes without linking them to anything.

### Solution
Every note should link to 2-3 related notes. Run `graph_isolated_nodes` weekly.

---

## Over-Linking

### Mistake
Linking every note to everything, creating a messy graph.

### Solution
Quality over quantity. Only link when there's meaningful connection.

---

## Skipping the Inbox

### Mistake
Trying to fully process notes at capture time.

### Solution
Capture quickly → Process later. Use an inbox.

---

## Not Reviewing

### Mistake
Building notes but never reviewing them.

### Solution
Per [[Self-Improvement Cycle]], run weekly health checks.
