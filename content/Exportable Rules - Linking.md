# Exportable Rules - Linking

> Drop into any CLAUDE.md for knowledge graph linking basics.

## Linking Principle
Every node should link to **2-3 related nodes**. Links must be content-driven, not random.

```markdown
# Good linking
[[Concept]] - brief explanation of how it connects

# Avoid
Links just to "see also" without context
```

**Demonstrated by:** `Linking Principle.md`

## Link Quality Rule
- Only link when there's meaningful connection
- Use descriptive link text: `[[Concept|Display Text]]`
- Prefer forward links; backlinks emerge naturally in Obsidian

**Demonstrated by:** `Linking Principle.md`

## Connection Types
Making meaningful connections:
- **Direct relevance** - Builds on each other
- **Contrast** - Opposition illuminates
- **Analogy** - Different domains, similar patterns
- **Cause/Effect** - One explains the other
- **Category** - Share common parent

Test: Can you explain WHY in one sentence?

**Demonstrated by:** `Linking Principle.md`
