# Building a Knowledge Base from Scratch

A step-by-step guide to starting your first knowledge graph. These principles work for any domain—software docs, research, personal knowledge, etc.

## Phase 1: Foundation (Week 1)

### Day 1-2: Setup
1. Install Obsidian or your preferred linked-note tool
2. Create your first note: `_root.md` with vault purpose
3. Add 3 foundational notes:
   - [[Atomic Note Principle]]
   - [[Linking Principle]]
   - [[Graph Traversal Efficiency]]

### Day 3-4: Capture
1. Process 1 existing document into atomic notes
2. Add frontmatter with tags
3. Link to foundational notes

### Day 5-7: First Links
1. Review notes and add 2-3 links each
2. Run `graph_hubs` to see connectivity
3. Fix any isolated notes

## Phase 2: Growth (Week 2-4)

### Weekly Routine
- **Daily:** Process 3-5 new atomic notes
- **Weekly:** Run graph health check
- **Monthly:** Review and update old notes

### Growth Metrics
| Week | Target Nodes |
|------|-------------|
| 1 | 10 |
| 2 | 25 |
| 3 | 50 |
| 4 | 100 |

## Phase 3: Maturity (Month 2+)

- Notes cross-link organically
- Emergence begins (new insights from connections)
- Exportable Rules documented
- Self-improvement cycle established

## Common Mistakes to Avoid

1. **Don't over-organize** - Links emerge, don't force
2. **Don't capture everything** - Process before saving
3. **Don't neglect maintenance** - Regular health checks
