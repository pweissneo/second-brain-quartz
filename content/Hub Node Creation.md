# Hub Node Creation

Hub nodes are highly connected notes that serve as entry points for broad topics in the knowledge graph.

## Purpose
- Provide starting points for exploring a topic
- Reduce navigation depth to any node
- Create logical clusters of related knowledge

## Characteristics
- High connectivity (many incoming and outgoing links)
- Broad topic coverage
- Clear, descriptive titles
- Links to sub-topic nodes

## Creation Guidelines

### When to Create
- New broad topic emerges that isn't covered
- Multiple notes refer to a similar theme without a central hub
- Topic grows beyond 5+ related notes

### How to Create
1. Identify the common theme
2. Create a named note with clear title
3. Link to existing notes in the topic area
4. Add to relevant parent hubs

## Example

```markdown
# Knowledge Management Methods

Entry point for PKM methodologies.

## Capture Methods
- [[Progressive Summarization]]
- [[Zettelkasten Method]]

## Practice Methods
- [[The Forgetting Curve]]
- [[Active Reading]]
- [[Writing as Thinking]]
```

*(Note: This example uses knowledge management concepts. Replace with examples from your domain.)*

## Maintenance
- Review hub nodes monthly
- Add new connections as knowledge grows
- Split if hub becomes too broad
