# Knowledge Base Bootstrap

Getting started with a new knowledge base in the first week.

---

## The Bootstrap Process (Week 1)

| Day | Action |
|-----|--------|
| 1 | Create `_root.md` (vault purpose) |
| 2 | Add Atomic Note Principle + Linking Principle |
| 3 | Process 1 document → 3-5 atomic notes |
| 4 | Link notes together |
| 5 | Run health check, fix issues |
| 6-7 | Repeat, aim for 10 notes |

**Target:** 10 notes → 25 → 50 → 100

---

## Quick Start Template

```markdown
# [Topic] Knowledge Base

## Purpose
[What does this vault cover?]

## Core Principles
- [[Atomic Note Principle]]
- [[Linking Principle]]

## Hub Notes
- [Topic] Hub → main entry point

## Daily Practice
- Capture → Process → Link → Review
```

---

## First Steps

1. **Define purpose** - One sentence in _root.md
2. **Add conventions** - Atomic, Linking, Navigation notes
3. **Process first content** - Start small, 3-5 notes
4. **Link intentionally** - Each note to 2-3 others
5. **Run health check** - Fix issues early
