# The Art of Connecting Notes

Making meaningful connections between ideas.

## Why Connections Matter

Connections create understanding:
- Single notes = facts
- Connected notes = knowledge
- Dense graph = insight

## What Makes a Good Connection?

### 1. Direct Relevance
Notes that explain or build on each other:
```
[[Moving Averages]] → [[MACD]] (MACD uses MA)
```

### 2. Contrast/Comparison
Notes that illuminate by opposition:
```
[[Bull Market]] → [[Bear Market]]
```

### 3. Analogy
Notes from different domains with similar patterns:
```
[[Ecosystems]] → [[Business Ecosystems]]
```

### 4. Cause/Effect
Notes where one explains the other:
```
[[Volatility]] → [[Risk Management]]
```

### 5. Category
Notes that share a common parent:
```
[[RSI]] → [[Momentum Indicators]]
```

## Anti-Patterns

### 1. False Connections
Linking because you "should" not because it makes sense.

### 2. Generic Links
"See also" without context.

### 3. Over-Connecting
Every note → every note.

## How to Find Connections

1. **Review neighbors** - What does this note connect to?
2. **Use graph_related** - Find 2-hop connections
3. **Write and ask** - "What does this remind me of?"
4. **Cross-pollinate** - Different domains connect unexpectedly

## The Connection Test

Can you explain WHY these notes connect in one sentence?
