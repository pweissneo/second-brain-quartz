# PARA Method

PARA is an organizational system for digital life developed by **Tiago Forte**. It organizes information by function, not topic.

## The Four Categories

| Category | Description | Examples |
|----------|-------------|----------|
| **P**rojects | Short-term outcomes | Blog post, app build, event planning |
| **A**reas | Ongoing responsibilities | Health, finances, career |
| **R**esources | Topics of interest | Recipes, research, reference |
| **A**rchives | Inactive items | Completed projects, old reference |

## PARA vs Zettelkasten

| Aspect | PARA | Zettelkasten |
|--------|------|--------------|
| Focus | Action-oriented | Knowledge-oriented |
| Structure | Hierarchy by domain | Network by connection |
| Use case | Personal productivity | Research & learning |
| Synergy | Works well together | Independent system |

## Integration with Knowledge Base Systems

PARA provides the **folder structure** while Zettelkasten provides the **linking logic**:

```
PARA (where)
├── Projects/
│   └── [Project notes with action items]
├── Areas/
│   └── [Area notes with ongoing focus]
├── Resources/
│   └── [Topic notes - these become Zettelkasten nodes]
└── Archives/
    └── [Completed/inactive]
```

Within Resources/, use [[Zettelkasten Method]] and [[Linking Principle]] to build the knowledge graph.
