# Self-Improvement Cycle

The knowledge base is a **living system** that continuously improves itself through structured iteration. This vault runs a feedback loop that diagnoses, course-corrects, and refills.

## Heartbeat Architecture (Current)

### The Feedback Loop
```
REFLECT --> fills backlog --> EXPLORE/AUDIT/CONNECT/DISTILL --> consumes backlog --> metrics update --> REFLECT evaluates --> adjusts priorities
```

### Modes

1. **REFLECT** - Diagnose vault health, course-correct problems, refill backlog
2. **EXPLORE** - Research and create new atomic notes
3. **AUDIT** - Verify existing notes against principles
4. **CONNECT** - Strengthen structural gaps in the graph
5. **DISTILL** - Extract patterns into Exportable Rules

### Decision Logic
- Backlog < 5 items → REFLECT
- Staleness is high → REFLECT  
- Consecutive no-commit >= 5 → REFLECT
- Otherwise → pop top backlog item

## Why This Works
- **Feedback loop** - Each iteration improves the next
- **Prioritization** - REFLECT ensures never stuck on low-value tasks
- **Metrics-driven** - Staleness detection catches stagnation
- **Self-referential** - This vault demonstrates its own methodology
